## 一、项目概述

### 1.1 游戏简介

乘法挑战是一款帮助儿童学习九九乘法表的教学小游戏。通过"展示 + 挑战 + 闯关"的渐进式玩法，让孩子在游戏中记住个位数乘法算式，完成全表背诵。

### 1.2 核心玩法设计思路

| 模式 | 名称 | 目标 | 说明 |
|------|------|------|------|
| 模式一 | **看一看** | 认知学习 | 展示完整的九九乘法表（三角形结构），点击任意算式可放大查看并听读音 |
| 模式二 | **填一填** | 练习记忆 | 显示一个乘法算式（如 3 × 4 = ?），用数字键盘填入答案 |
| 模式三 | **闯一闯** | 通关挑战 | 5 个关卡，从易到难，每关 5 题，连续答对通关 |

### 1.3 教学价值

- 认识乘法符号（×）和等号（=）
- 学会 1~9 的个位数乘法
- 掌握九九乘法表 45 个核心算式
- 培养数感和快速心算能力

    ↓

### 6.7 自动提交逻辑

- **1 位数答案**（1~9）：输入后 0.5 秒自动提交
- **2 位数答案**（10~81）：输入第二位后自动提交
- 也可点击"✓ 提交"按钮手动提交
- 避免误判：答案如果是 1~9，但孩子先输入了 0（如 `06`），自动去掉前导 0

## 八、题目生成算法（核心逻辑）

### 8.1 题目生成

```javascript
// 随机生成符合难度的乘法题
function generateQuestion(difficulty) {
    let a, b;

    switch (difficulty) {
        case 'easy':   // 1~5 的乘法
            a = random(1, 5);
            b = random(1, 5);
            break;
        case 'medium': // 1~7 的乘法
            a = random(1, 7);
            b = random(1, 7);
            break;
        case 'hard':   // 1~9 的乘法
            a = random(1, 9);
            b = random(1, 9);
            break;
    }

    // 保证 a ≤ b（符合九九乘法表习惯：小数在前）
    if (a > b) [a, b] = [b, a];

    return { a, b, answer: a * b };
}
```

### 8.2 闯关模式关卡分配

```javascript
// 第 n 关的题目生成规则
function generateLevelQuestion(level) {
    switch (level) {
        case 1: // 1 × (1~9)
            return { a: 1, b: random(1, 9), answer: b };
        case 2: // 2 × (1~9)
            return { a: 2, b: random(1, 9), answer: 2 * b };
        case 3: // 3~5 的乘法
            return randomFromRange(3, 5);
        case 4: // 6~7 的乘法
            return randomFromRange(6, 7);
        case 5: // 8~9 的乘法
            return randomFromRange(8, 9);
    }
}
```

### 8.3 保证题目不重复

- 使用已答题目集合 `answeredSet`
- 生成新题时检查是否在集合中
- 如果在集合中，重新生成
- 当集合接近满时（如 40 个题目已答过），清空集合

## 十四、异常处理与边界情况

### 14.1 输入异常

| 场景 | 处理方式 |
|------|---------|
| 孩子连续输入超过答案位数 | 自动忽略多余输入 |
| 输入了 0 开头的答案（如 06） | 自动去除前导 0，视为 6 |
| 连续输入相同数字 | 正常处理（如答案 66 可输入两个 6） |
| 删除键按下 | 清空最后一个输入数字，如为空则无操作 |

### 14.2 数据异常

| 场景 | 处理方式 |
|------|---------|
| LocalStorage 不可用 | 使用内存变量存储，提示"数据仅本次有效" |
| 读取到格式错误数据 | 使用默认数据初始化 |
| 版本不匹配 | 迁移数据或重置 |

### 14.3 特殊算式边界

| 场景 | 说明 |
|------|------|
| 1 × 1 = 1 | 答案 1 位数，需正确处理 |
| 9 × 9 = 81 | 答案 2 位数，两位答案的首位输入 |
| 答案为 0 | 九九乘法表中不会出现 0，无需处理 |
| 算式 a ≤ b | 保证第一个数 ≤ 第二个数（符合小学生学习习惯） |

## 十八、关键设计决策记录

| 决策项 | 决策内容 | 理由 |
|--------|---------|------|
| 算式排列 | 三角形（1×1，1×2/2×2，...） | 符合传统九九乘法表的视觉习惯 |
| 数字色彩 | 1~9 映射 9 种不同糖果色 | 增加视觉辨识度，儿童更容易记住 |
| a × b 顺序 | 保证 a ≤ b（小数在前） | 符合小学生学习习惯（2×3 而非 3×2） |
| 自动提交 | 填满答案位数后自动提交 | 减少操作步骤，提升游戏流畅度 |
| 关卡数量 | 5 关（递增难度） | 不太多也不太少，保持挑战感 |
| 每关题目数 | 5 题 | 快速通关，保持成就感 |
